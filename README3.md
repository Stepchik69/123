Вариант №4
Общее описание алгоритма
Алгоритм основан на принципе рекурсии и работает следующим образом:
Базовый случай: если строка пуста или содержит один символ, возвращаем её как есть
Рекурсивный случай: берём последний символ строки и добавляем к результату рекурсивного вызова функции для оставшейся части строки (без последнего символа)
Процесс продолжается до достижения базового случая

Временная сложность
O(n), где n — длина строки

Объяснение временной сложности
На каждом шаге рекурсии:
Выполняется константное количество операций (доступ к символу, конкатенация)
Длина обрабатываемой строки уменьшается на 1
Количество рекурсивных вызовов равно длине исходной строки
Таким образом, общее количество операций линейно зависит от длины строки.

Контрольный вопрос №7: Временная сложность задачи о N ферзях
O(N!), где N — размер доски. Это объясняется тем, что:

На каждой строке нужно попробовать разместить ферзя в каждом столбце

При этом происходит проверка всех предыдущих позиций

В худшем случае алгоритм перебирает все возможные расстановки, количество которых растёт факториально с увеличением N
